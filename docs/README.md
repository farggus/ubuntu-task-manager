# Documentation Guidelines

All documentation in this project **MUST** be written in **English**.

## Why English?

This is an international open-source project. English documentation ensures:
- Global accessibility for all contributors and users
- Easier code reviews and collaboration
- Professional standards
- Broader community participation

## What Goes Here

This directory contains technical documentation, design decisions, and guides:

- **Architecture documents** - System design and component interactions
- **ADRs (Architecture Decision Records)** - Why certain technical decisions were made
- **Development guides** - How-to guides for specific tasks
- **Refactoring notes** - Documentation of major code changes

## Current Documentation

- [`logging_refactoring.md`](./logging_refactoring.md) - Fail2Ban logging improvements (Jan 2026)

## Contributing Documentation

When adding documentation:

1. **Use English** - All text must be in English
2. **Be clear and concise** - Avoid jargon when possible
3. **Include examples** - Code examples help understanding
4. **Keep it updated** - Update docs when code changes
5. **Link related docs** - Cross-reference related documentation

### Template for New Docs

```markdown
# [Feature/Component Name]

## Overview
Brief description of what this document covers.

## Background
Context and motivation for this work.

## Changes/Decisions
What was changed or decided, and why.

## Examples
Code examples showing usage.

## References
Links to related docs, issues, or PRs.
```

## Language Policy

> ⚠️ **ALL** documentation must be in English, including:
> - Markdown files
> - Code comments
> - Commit messages
> - Diagrams and charts
> - Examples and snippets

This is a strict requirement for all contributions.

See [CONTRIBUTING.md](../CONTRIBUTING.md) for full guidelines.
